# LLM Evaluation Notebook

A practical walkthrough of how to evaluate LLM output quality using the RAGAS framework.

Most teams ship AI features and hope they work. This notebook shows how to actually measure it.

---

## What this covers

- **Faithfulness**: Did the AI's answer come from the source, or did it hallucinate?
- **Answer Relevance**: Did it actually answer what was asked?
- **Context Recall**: Did the retrieval system surface the right information?

---

## Why this matters

In enterprise AI, a wrong answer isn't just a bad user experience. It's a trust problem, a compliance risk, and a product failure. Evaluation frameworks like RAGAS give you the instrumentation to catch these issues before users do.

This is based on real evaluation work I did building enterprise RAG pipelines at AriesView.

---

## How to run it

1. Clone this repo or open the notebook in Google Colab
2. Add your OpenAI API key where indicated
3. Run cells top to bottom
